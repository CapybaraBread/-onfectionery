# Сладкий сундук

Сайт кондитерской, который создается из Excel-данных через Jinja и запускается встроенным Python HTTP-сервером.

## Требования

- Python 3.8+
- Jinja2
- openpyxl

Установка зависимостей:

```bash
python3 -m pip install -r requirements.txt
```

## Настройка отправки писем

Render не предоставляет доступ к Gmail SMTP в текущей среде, поэтому заявки
отправляются через HTTPS API сервиса Resend.

1. Зарегистрируйтесь на <https://resend.com> с адресом `denis05102012@gmail.com`.
2. Создайте API-ключ в разделе API Keys.
3. Локально задайте ключ перед запуском:

```bash
export RESEND_API_KEY="re_ваш_ключ"
```

4. На Render добавьте Environment Variable:

```text
RESEND_API_KEY=re_ваш_ключ
```

После сохранения переменной выполните новый deploy. Секретный ключ нельзя
записывать в Python, README или загружать на GitHub. Старую переменную
`GMAIL_APP_PASSWORD` можно удалить.

## Запуск

```bash
python3 render_website.py
```

Скрипт:

1. Читает `site_data.xlsx`.
2. Создает `index.html` из Jinja-шаблонов.
3. Запускает сайт на <http://127.0.0.1:8000>.
4. Принимает форму на `/send-phone` и отправляет номер телефона на почту.

Остановка сервера: `Ctrl+C`.

## Изменение содержимого

Тексты, товары, отзывы и галерея хранятся в `site_data.xlsx`. После изменения таблицы перезапустите сервер, чтобы заново создать `index.html`.

Основные файлы:

- `render_website.py` — генерация страницы, сервер и отправка писем;
- `templates/index.html` — основной Jinja-шаблон;
- `templates/macros.html` — шаблоны товара и отзыва;
- `site_data.xlsx` — данные сайта;
- `style.css` и `script.js` — оформление и интерактивность;
- `data/` — изображения.

## Важно

Отправка писем не будет работать при простом открытии `index.html` или публикации только на GitHub Pages. Нужен запущенный `render_website.py` на сервере с переменной `RESEND_API_KEY`.
