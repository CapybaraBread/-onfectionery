# Сладкий сундук

Сайт кондитерской на Python. Данные товаров, отзывов и галереи берутся из
`site_data.xlsx`, после чего Jinja создает готовый `index.html`.

`render_website.py` одновременно:

1. Генерирует `index.html`.
2. Запускает локальный HTTP-сервер.
3. Принимает заявки формы по адресу `/send-phone`.
4. Отправляет номер телефона на `denis05102012@gmail.com` через Resend API.

## Структура проекта

- `render_website.py` - генерация страницы, сервер и обработка формы;
- `templates/index.html` - основной Jinja-шаблон;
- `templates/macros.html` - шаблоны карточек товаров и отзывов;
- `site_data.xlsx` - тексты и данные сайта;
- `style.css` - оформление сайта;
- `script.js` - слайдер, модальное окно и отправка формы;
- `data/` - изображения;
- `index.html` - автоматически созданная страница;
- `requirements.txt` - зависимости Python.

## Подготовка

Требуется Python 3.8 или новее.

Перейдите в папку проекта:

```bash
cd "/home/denis/YandexDisk/projects all/Сonfectionery"
```

Установите зависимости:

```bash
python3 -m pip install -r requirements.txt
```

При желании можно использовать виртуальное окружение:

```bash
python3 -m venv .venv
source .venv/bin/activate
python3 -m pip install -r requirements.txt
```

## Настройка Resend

Render не может подключиться к Gmail SMTP из текущей среды. Поэтому письма
отправляются через Resend по HTTPS.

1. Зарегистрируйтесь на <https://resend.com> с адресом
   `denis05102012@gmail.com`.
2. Откройте <https://resend.com/api-keys>.
3. Создайте ключ с правом отправки писем.
4. Скопируйте ключ сразу после создания.

Правильный ключ Resend начинается с:

```text
re_
```

Ключ Render начинается с `rnd_` и для отправки писем не подходит.

Никогда не добавляйте настоящий API-ключ в Python, README или GitHub.

## Локальный запуск

В том же терминале задайте ключ Resend:

```bash
export RESEND_API_KEY="re_ваш_настоящий_ключ"
```

Запустите сайт:

```bash
python3 render_website.py
```

Откройте:

```text
http://127.0.0.1:8000
```

Не открывайте `index.html` двойным кликом, если хотите проверить форму.
При таком открытии серверный адрес `/send-phone` работать не будет.

Остановка сервера:

```text
Ctrl+C
```

Если порт `8000` занят, найдите ранее запущенный терминал с сайтом и
остановите процесс сочетанием `Ctrl+C`.

### Повторный локальный запуск

Переменная, заданная через `export`, действует только в текущем терминале.
После открытия нового терминала выполните ее снова:

```bash
export RESEND_API_KEY="re_ваш_настоящий_ключ"
python3 render_website.py
```

## Запуск на Render

### 1. Отправка кода в GitHub

После изменений выполните:

```bash
git add .
git commit -m "Описание изменений"
git push origin main
```

Не используйте `git add .`, если в проекте есть файлы, которые не должны
попасть в коммит. В таком случае добавляйте нужные файлы отдельно.

### 2. Настройки сервиса

В Render должны быть указаны:

```text
Build Command: pip install -r requirements.txt
Start Command: python3 render_website.py
```

Текущий сервер запускается на порту `8000`, поэтому в разделе Environment
оставьте:

```text
PORT=8000
```

### 3. Переменная Resend

Откройте:

```text
Render -> Confectionery -> Environment
```

Добавьте:

```text
RESEND_API_KEY=re_ваш_настоящий_ключ
```

Значение вводится без кавычек и лишних пробелов. Переменные
`GMAIL_APP_PASSWORD` и ключи Render вида `rnd_...` для формы не нужны.

После изменения переменных нажмите `Save Changes`. Затем дождитесь
автоматического deploy или выберите:

```text
Manual Deploy -> Deploy latest commit
```

## Диагностика ошибок

Логи находятся здесь:

```text
Render -> Confectionery -> Logs
```

Основные ошибки:

- `На сервере не настроена отправка почты` - отсутствует
  `RESEND_API_KEY`.
- `401 API key is invalid` - указан неверный ключ. Нужен ключ Resend,
  начинающийся с `re_`.
- `403 error code: 1010` - запущена старая версия кода без заголовка
  `User-Agent`. Проверьте, что последний коммит попал в GitHub и Render.
- `502` - подробная причина находится рядом в Render Logs.
- Письмо отправлено, но его нет во входящих - проверьте папку «Спам» и
  журнал Emails в панели Resend.

## Изменение содержимого

Товары, отзывы, галерея и основные тексты находятся в `site_data.xlsx`.
После изменения таблицы перезапустите `render_website.py`. Скрипт заново
создаст корневой `index.html`.

Шаблон страницы изменяется в `templates/index.html`, а повторяющиеся
карточки - в `templates/macros.html`.

Не редактируйте сгенерированный `index.html` как основной источник:
при следующем запуске скрипта он будет перезаписан.

## Важно

- Форма работает только при запущенном `render_website.py`.
- GitHub Pages не умеет выполнять этот Python-код.
- Для публикации используется Render Web Service.
- Секретные ключи хранятся только в переменных окружения.
- После изменения кода требуется `git push`, а после изменения переменных
  Render - новый deploy.
